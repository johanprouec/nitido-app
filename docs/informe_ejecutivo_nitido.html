01 · Contexto

El problema no es solo predecir: es poder defender la decisión.

NÍTIDO quiere usar inteligencia artificial para hacer un primer filtro de hojas de vida. El valor de negocio es claro: reducir carga operativa y priorizar candidatos para revisión humana. El riesgo también es claro: una decisión automatizada sobre personas puede generar daño reputacional, legal y regulatorio si no se explica ni se audita.

Trabajamos con 5.000 casos históricos y 18 variables anonimizadas. La variable objetivo indica si un candidato avanzó o no a entrevista. Como los nombres reales de las variables no fueron entregados, el análisis se concentró en el comportamiento del modelo y en los riesgos de usar señales que podrían representar características sensibles de forma indirecta.

El objetivo de este informe no es mostrar código. El objetivo es responder si el sistema puede usarse, bajo qué condiciones y qué controles debe exigir NÍTIDO antes de venderlo a clientes regulados.

02 · Qué encontramos

El modelo tiene señal útil, pero todavía necesita control humano.

El modelo final es una regresión logística baseline con umbral 0.45. En test obtuvo F1-score 0.7241 y AUC 0.8056, evidencia suficiente para justificar un piloto supervisado. Sin embargo, sus decisiones dependen con fuerza de algunas variables anonimizadas. Eso obliga a una revisión interna antes de usarlo como sistema definitivo.

Hallazgo 1

El sistema aporta valor como herramienta de apoyo operativo.

El modelo permite ordenar candidatos, identificar casos con mayor prioridad y explicar qué señales empujan una decisión hacia avanzar o no avanzar. Es útil como apoyo para equipos de selección.

Hallazgo 2

Hay una zona gris que no debe automatizarse.

Los candidatos cerca del punto de decisión pueden cambiar de resultado con variaciones mínimas. Esos casos deben pasar a revisión humana obligatoria, no a descarte automático.

Hallazgo 3

No se puede declarar libre de sesgo.

No se identificó discriminación directa porque las variables están anonimizadas. Aun así, no sabemos si las más influyentes son legítimas o si funcionan como sustitutos de edad, género, ubicación, institución educativa o nivel socioeconómico.

03 · Riesgos críticos

El principal riesgo no está en la tecnología, sino en usarla sin gobernanza.

El análisis de explicabilidad mostró que algunas variables dominan las decisiones. Las variables más importantes fueron x1, x2, x5, x9, x7, x10, x4, x3 y x14. En particular, x1, x2 y x14 requieren mayor atención de auditoría antes de producción.

Riesgo regulatorio

Variables desconocidas.

No se afirma discriminación directa, porque las variables están anonimizadas y no revelan atributos sensibles. El riesgo es de proxy sensible: sin conocer el significado real de las variables, NÍTIDO no puede descartar que alguna señal actúe como sustituto indirecto de un atributo protegido.

Riesgo histórico

Variables dominantes.

x1 y x2 dominan la decisión del modelo. Si estas variables representan atributos sensibles o proxies de atributos sensibles, el sistema podría generar riesgo regulatorio o ético aunque sus métricas generales sean buenas.

Riesgo operativo

Efecto negativo de x14.

x14 tiene un efecto negativo: valores altos reducen la probabilidad estimada de avanzar. Esta variable debe revisarse con especial cuidado para confirmar que su uso sea legítimo, explicable y proporcional.

04 · Evidencia visual

Lo que muestran las gráficas del análisis.

Estas visualizaciones no buscan llenar el informe de técnica. Sirven para respaldar tres ideas ejecutivas: el modelo sí tiene señal, el corte de decisión debe manejarse con cuidado y las variables dominantes requieren auditoría.

Base histórica de decisiones

El sistema aprende de decisiones pasadas. Esto permite capturar patrones reales del proceso, pero también exige verificar que el histórico no haya incorporado sesgos.

El modelo final se eligió con criterio

La regresión logística baseline fue seleccionada como modelo final porque obtuvo el mejor equilibrio entre desempeño predictivo e interpretabilidad. En validación presentó el mayor F1-score y el mayor AUC frente a Random Forest y Gradient Boosting. Además, permite explicar mejor la dirección de los efectos, justificar el umbral operativo y auditar las decisiones del sistema. El único artefacto final exportado es models/modelo_nitido.pkl.

El error sensible es descartar buenos candidatos

La matriz ayuda a ver dónde falla el sistema. En selección, el riesgo más delicado es dejar por fuera perfiles que merecían revisión humana.

El modelo separa mejor que una decisión aleatoria

La curva respalda que existe capacidad de priorización. Aun así, no elimina la necesidad de controles en casos cercanos al umbral.

Las variables dominantes deben ser auditadas

La explicación global muestra que x1, x2, x5, x9, x7, x10, x4, x3 y x14 influyen más que el resto. Como sus nombres están ocultos, NÍTIDO debe revisar internamente si son legítimas o si pueden operar como sustitutos de características sensibles.

Lectura ejecutiva: las gráficas respaldan una salida condicionada. El modelo tiene utilidad para priorizar candidatos, con AUC test de 0.8056 y F1-score test de 0.7241, pero todavía necesita auditoría de variables, revisión humana en zona gris y monitoreo continuo.

06 · Recomendación final

Avanzar con piloto controlado, no con producción plena.

Sí, pero no en automático. Se recomienda avanzar a un piloto controlado con revisión humana obligatoria, monitoreo de métricas y auditoría interna del significado de las variables. No se recomienda usar el modelo como mecanismo de descarte automático.

“NÍTIDO puede mostrar que tiene un sistema útil, pero todavía no debe venderlo como una máquina que decide sola sobre personas.”

07 · Próximos pasos

Qué debe hacer NÍTIDO antes de venderlo a un banco.

La salida responsable exige controles concretos. Estos pasos convierten el prototipo en un sistema defendible ante clientes, reguladores y comités internos.

01

Revisar internamente el significado de las variables críticas.

NÍTIDO debe levantar la anonimización dentro de su equipo autorizado y verificar que x1, x2, x5, x9, x7, x10, x4, x3 y x14 sean legítimas para selección. La revisión prioritaria debe enfocarse en x1, x2 y x14.

02

Medir resultados por grupos reales.

Antes de producción, se deben comparar tasas de aprobación, errores y rechazos por grupos relevantes, respetando la ley y la política de datos.

03

Enviar casos cercanos a revisión humana.

Los casos en zona gris no deben rechazarse automáticamente. Deben quedar en una cola especial con explicación de factores y decisión documentada por un humano.

04

Crear bitácora y monitoreo continuo.

Cada decisión debe dejar trazabilidad: probabilidad, explicación, cambios sugeridos y decisión final. Además, el desempeño debe revisarse periódicamente para detectar deterioro o sesgos nuevos.

05

Comunicar el alcance sin exagerar.

El mensaje comercial debe ser prudente: NÍTIDO no ofrece un reemplazo del reclutador, sino una herramienta para priorizar, explicar y apoyar decisiones humanas.
